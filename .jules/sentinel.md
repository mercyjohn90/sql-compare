## 2024-05-20 - difflib.HtmlDiff Metadata XSS and Variable Shadowing
**Vulnerability:** XSS in HTML report generation because dynamically generated text (e.g., summary bullet points) and metadata inputs (e.g., `fromdesc`, `todesc`) passed to `difflib.HtmlDiff.make_table` were not HTML escaped.
**Learning:** `difflib.HtmlDiff` escapes the code chunks being compared, but it explicitly does **not** escape metadata fields like `fromdesc` and `todesc`. Furthermore, when adding `import html` to a file to use `html.escape()`, local variables named `html` will shadow the module name, causing runtime `NameError` or `AttributeError` exceptions if not renamed to something else (e.g., `html_content`).
**Prevention:** Always explicitly use `html.escape()` on any strings injected into HTML templates or passed as metadata into templating engines/libraries that don't guarantee full auto-escaping. Also, strictly avoid using standard library module names (like `html`) as local variable names.
